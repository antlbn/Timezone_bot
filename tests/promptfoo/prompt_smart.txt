ВЫВОДИ ТОЛЬКО JSON. ОТВЕТ НАЧИНАЕТСЯ С { И ЗАКАНЧИВАЕТСЯ НА }.

ЗАДАЧА:
Проанализируй CURRENT MESSAGE с учетом PREVIOUS CHAT CONTEXT.
Цель: извлечь информацию о планируемых/обсуждаемых событиях (встречи, созвоны, дедлайны) и их времени.

ИНСТРУКЦИИ:
1. Выводи время строго в 24-часовом формате ("HH:MM").
2. Инферируй AM/PM из контекста и времени сообщения (например, рабочие часы).
3. Учитывай языковые идиомы времени: "пол восьмого" = 07:30 или 19:30, "без двадцати пять" = 04:40 или 16:40. Но "в 10 утра" = 10:00 (это не идиома "пол").
4. Относительное время ("через 10 минут") вычисляй от timestamp текущего сообщения.
5. Если упомянуто одно и то же время в разных часовых поясах, выбери только одно значение для вывода (чтобы не было дублей) и соответствующий ему город.

JSON SCHEMA:
{
  "reflections": {
    "event_logic": "string (краткая суть события)",
    "time_logic": "string (логика определения времени в 24h формате)",
    "geo_logic": "string (город или часовой пояс, если указано)"
  },
  "event": boolean (true, если есть координация, договорённость или дедлайн с временем),
  "time": ["HH:MM"],
  "city": [string | null] (массивы time и city одной длины)
}

ПРИМЕРЫ:

Пример 1 (Идиомы и контекст):
- 12:00 John: "Завтра в пол десятого соберёмся"
-> {"reflections":{"event_logic":"планируется встреча на завтра","time_logic":"'пол десятого' = 09:30 или 21:30. Из рабочего контекста вероятнее 09:30","geo_logic":"не указан"},"event":true,"time":["09:30"],"city":[null]}

Пример 2 (Перевод времени в разных зонах):
- 18:30 Jane: "не вечером - утром: в 8 по Парижу это в 11 по Еревану"
-> {"reflections":{"event_logic":"утренняя встреча","time_logic":"8 по Парижу = 11 по Еревану. Оставляю одно переведенное значение","geo_logic":"Ереван"},"event":true,"time":["11:00"],"city":["Erevan"]}

Пример 3 (Флуд / Отсутствие времени события):
- 21:22 Оля: "ребят вы серьезно? У нас есть чат для флуда"
-> {"reflections":{"event_logic":"флуд, без организационных вопросов","time_logic":"время не упоминается","geo_logic":"не указан"},"event":false,"time":[],"city":[]}
