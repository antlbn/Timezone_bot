# Promptfoo: Команды и Логика

`promptfoo` — это инструмент для тестирования (eval) выходов LLM. В вашем проекте он используется для проверки того, как модели (Gemma, Llama) извлекают время из сообщений.

## Основные команды

### 1. `promptfoo eval`
Основная команда для запуска тестов. Она прогоняет ваши промпты через указанные провайдеры на наборе тестовых кейсов.

*   **Логика**: Читает конфиг -> Отправляет запросы в LLM -> Проверяет ответы (assert) -> Сохраняет результат в локальную БД (`promptfoo/output`).

### 2. `promptfoo view`
Запускает локальный веб-интерфейс (обычно на `localhost:15500`) для удобного просмотра результатов.
*   Там можно увидеть таблицу сравнения, JSON ответы и причины провала тестов.

### 3. `promptfoo share`
Создает временную публичную ссылку на ваши результаты (удобно, чтобы показать кому-то отчет).

## Ключевые Аргументы

| Аргумент | Описание | Зачем нужен вам? |
| :--- | :--- | :--- |
| **`-c, --config <path>`** | Путь к файлу конфигурации. | Если вы запускаете команду из корня проекта, нужно указать путь к [promptfooconfig.yaml](file:///Users/johnwunderbellen/Timezone_bot/tests/promptfoo/promptfooconfig.yaml). |
| **`--filter-providers <regex>`** | Запуск тестов только для конкретных моделей. | Позволяет не тратить время/деньги на все модели, а запустить, например, только `groq`. |
| **`--env-file <path>`** | Путь к [.env](file:///Users/johnwunderbellen/Timezone_bot/.env) файлу с ключами. | Чтобы `promptfoo` увидел ваш `GROQ_API_KEY`. |
| **`-o, --output <file>`** | Сохранить результат в файл (json/html/csv). | Для автоматизации или сохранения логов. |

## Примеры для вашего проекта

### Запуск всего из корня:
```bash
npx promptfoo eval -c tests/promptfoo/promptfooconfig.yaml --env-file .env
```

### Запуск только для Groq:
```bash
npx promptfoo eval -c tests/promptfoo/promptfooconfig.yaml --filter-providers groq --env-file .env
```

### Просмотр результатов:
```bash
npx promptfoo view
```

### Важные нюансы
1. **Пути в конфиге**: Помните, что `promptfoo` считает пути относительно файла конфигурации. Если в конфиге написано `file://prompt.txt`, то файл должен лежать в той же папке, что и конфиг.
2. **Кэширование**: По умолчанию `promptfoo` кэширует ответы. Если вы не меняли промпт или переменные, он может не делать новый запрос. Чтобы форсировать запуск, используйте `--no-cache`.
