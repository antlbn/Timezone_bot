ВЫВОДИ ТОЛЬКО JSON. ОТВЕТ НАЧИНАЕТСЯ С { И ЗАКАНЧИВАЕТСЯ НА }.

Анализируй CURRENT MESSAGE (с учётом PREVIOUS CHAT CONTEXT). Найди будущие/текущие события с временными метками — звонки, встречи, дедлайны, договорённости.

ПРАВИЛА ВРЕМЕНИ (24ч формат):
- «Пол [час]а» → :30 предыдущего часа (пол восьмого = 07:30 или 19:30)
- «Без [X] [час]» → час минус X минут (без пятнадцати восемь = 07:45 или 19:45)
- «[X] минут [час]ого» → предыдущий час + X минут (двадцать минут шестого = 05:20)
- «Half past [h]» → h:30 | «Quarter to [h]» → (h-1):45 | «Quarter past [h]» → h:15
- «halb [h]» (нем.) → (h-1):30 (halb zehn = 09:30, не 10:30!)
- «[h] y media» (исп.) → h:30 | «[h]h» (фр.) → h:00
- Относительное время считай от timestamp сообщения («через час» от 12:21 = 13:21)
- «чз [N] мин» / «через [N] минут» → timestamp + N минут
- Если AM/PM не указан — инферируй из времени сообщения и рабочего контекста (если timestamp > 12:00 и нет явного «утра» → предпочитай PM)
- «В час» / «в два» в рабочем контексте → предпочитай дневное время (13:00, 14:00), не ночное
- «После обеда» / «утром» / «вечером» / «попозже» без конкретного числа → НЕ записывай в time, но используй как подсказку AM/PM для числа рядом («после обеда, say в 4» → 16:00)
- ТОЛЬКО слово «пол» запускает правило −30 минут:
  ✓ «пол десятого» → 09:30 (идиома)
  ✗ «в десять утра» → 10:00 (обычный AM-модификатор, НЕ идиома!)
  ✗ «в восемь вечера» → 20:00 (обычный PM-модификатор, НЕ идиома!)

ПРАВИЛА КОНТЕКСТА:
- Числа в тексте — сначала проверь: не является ли число номером подъезда, этажа, задачи, номера здания
- Прошедшее время («вчера в 9», «было в 11») → событие уже случилось, но может быть релевантным если явно планируется повтор или это дедлайн
- Если автор явно исправляет время («не в 10, а в 9:30» / «not morning, 7pm») → бери исправленное, игнорируй отменённое
- История нужна только для контекста текущего сообщения; не извлекай события только из истории

ПРАВИЛА EVENT:
- event: true → есть координация, договорённость, дедлайн с временем (даже неточным)
- event: false → флуд, личные планы без координации, просто чат без события

SCHEMA:
{
  "reflections": {
    "event_logic": "суть события и нужна ли координация",
    "time_logic": "вывод времени шаг за шагом, 24ч",
    "geo_logic": "город или таймзона если упомянуты"
  },
  "event": boolean,
  "time": ["HH:MM"],
  "city": [string | null]
}
