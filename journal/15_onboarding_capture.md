# 15. Перехват сообщений при онбординге (Zero-Friction Onboarding)

## Проблема
Когда новый пользователь (без установленного города/таймзоны) отправляет сообщение с упоминанием времени, бот прерывает стандартный пайплайн обработки для проведения онбординга. Ранее исходное сообщение просто отбрасывалось, что заставляло пользователя повторять свой запрос после настройки профиля.

## Решение
Внедрен механизм временного сохранения "подвешенных" (pending) сообщений, который позволяет автоматически обработать их сразу после завершения онбординга.

### 1. Временное хранилище (Redis)
Для хранения используется Redis (база 1 по умолчанию).
- **Ключ:** `pending_msg:{platform}:{user_id}`
- **TTL:** 60 секунд.
- **Данные:** JSON-объект, содержащий текст сообщения, `chat_id`, `message_id`, `author_name`, `timestamp_utc` и **Snapshot** (снимок истории чата).

### 2. Сохранение контекста (Snapshot)
Чтобы LLM корректно поняла сообщение спустя время (когда история чата могла убежать вперед), в момент первого триггера бот делает снимок последних N сообщений.
- Этот снимок сохраняется в Redis вместе с сообщением.
- При восстановлении сообщения этот снимок передается в `process_message` через параметр `precomputed_snapshot`.

### 3. Обработка блокировок чата
В ядре `process_message` реализована дифференцированная логика блокировок:
- **Обычные сообщения:** Используют fail-fast. Если чат занят обработкой другого запроса, новое сообщение игнорируется.
- **Восстанавливаемые сообщения:** Теперь **ждут** освобождения лока (`async with lock`). Это гарантирует, что пользователь, прошедший онбординг, обязательно получит ответ на свое исходное сообщение, даже в активном чате.

### 4. Визуальная связь (Reply)
После настройки города бот отвечает на исходное сообщение пользователя с помощью механизма **Reply** (Telegram) или **Reference** (Discord). Это позволяет пользователю четко видеть, на какой именно его вопрос пришел ответ.

## Компоненты
- **Storage:** `src/storage/pending.py`
- **Core logic:** `src/event_detection/__init__.py` (параметры `skip_history_append`, `precomputed_snapshot`).
- **Telegram:** `src/commands/common.py` (захват), `src/commands/settings.py` (обработка).
- **Discord:** `src/discord/events.py` (захват), `src/discord/commands.py` (обработка).
